# 多代理人交易分析 (全面核准工作流)

## 概述
本技能引導一個專業等級的股票分析流程。核心特點在於「用戶主導的內容篩選與流程控制」。每個階段的代理人輸出都必須經過用戶審核，用戶決定哪些觀點被採納、哪些被剔除，或是否需要重新進行研究/辯論。

## 執行流程

### 第一階段：專家研究與用戶確認
1. **數據獲取**：使用 Yahoo Finance MCP 工具並行獲取財務、新聞、宏觀指標、價格歷史及期權鏈數據。
2. **專家報告**：調用 `fundamental-analyst`, `technical-analyst`, `sentiment-analyst`, `macro-analyst`, `options-analyst` 及 `portfolio-analyst` 生成報告。
3. **用戶審核 (關鍵互動點)**：呈現六份報告（包含宏觀、期權及投資組合分析）。
   - **用戶操作**：決定是否接受報告、指定哪些部分不合理需剔除、或要求針對特定項目重跑。
   - **輸出**：僅保留「用戶確認核准」的內容進入下一階段。

### 第二階段：牛熊立場辯論
1. **初始化辯論**：將「用戶確認核准」的專家報告交給 `bullish-analyst` 與 `bearish-analyst`。
2. **執行辯論**：代理人嚴格且僅能基於已核准的內容進行攻防。
3. **用戶辯論審核 (關鍵互動點)**：展示辯論報告。
   - **用戶操作**：決定是否接受辯論結果、篩選需要保留的攻防點、或要求重新進行更深層次的辯論。
   - **輸出**：產生「用戶確認核准」的辯論總結。

### 第三階段：交易策略與量化驗證
1. **策略生成**：調用 `trader` 代理人。
2. **量化回測與風險分析**：在 `trader` 生成策略後，自動執行以下動態分析：
   - **動態回測**：計算包含交易成本與滑價的歷史績效。
   - **壓力測試**：執行 99% VaR/CVaR 計算及 10,000 次蒙地卡羅路徑模擬。
3. **輸入限制**：分析師僅能接收第一階段核准的報告內容及第二階段核准的辯論總結。
4. **策略、回測與風險審核 (關鍵互動點)**：展示交易策略建議、對應的回測報告及壓力測試數據。
   - **用戶操作**：審核點位、績效指標與風險機率，決定是否採納、修改建議、或要求重新構思。

### 第四階段：風險管理團隊辯論
1. **團隊介入**：調用 `aggressive-risk-manager`, `neutral-risk-manager`, `conservative-risk-manager`。
2. **輸入限制**：風險團隊僅能基於「用戶確認核准」的所有前序內容進行評估。
3. **分輪辯論與審核 (關鍵互動點)**：
   - 每一輪辯論結束後，必須向用戶展示三方的核心觀點。
   - **用戶操作**：針對每一輪內容決定是否接受、剔除不合理觀點、或要求繼續下一輪/重跑本輪。
   - **輸出**：產生最終「用戶確認核准」的風險攻防紀錄。

### 第五階段：自動決策與最終報告
1. **最終決策**：調用 `final-manager`。
2. **輸入限制**：`final-manager` 整合所有前序階段「用戶確認核准」的內容。
3. **任務**：自動做出 EXECUTION 或 REJECT 決策，並生成一份嚴格基於核准內容的專業投資報告。

## 核心準則
- **內容溯源**：嚴禁代理人引入未經用戶核准或專家報告中未提及的外部資訊。
- **強制停頓**：在所有「關鍵互動點」必須停止，等待用戶明確指示「接受、修改或重跑」後方可繼續。
- **一致性**：所有階段的輸出必須與前序階段用戶核准的內容保持嚴格一致。
- **語言**：所有輸出必須使用繁體中文。
